Quink

Quink is an HTML editor that can be added to any web page in order to allow users to edit content on that page. Quink makes use of the contenteditable feature in HTML5 to edit document content.

There are two ways to invoke editing commands in Quink:

  • the editing toolbar
  • the keyboard

Quink's editing toolbar is accessed by double tapping on an editable region. As well as the toolbar, Quink provides a set of keyboard bindings which can be used to invoke editing commands. These are available when Quink is set into command mode. Quink starts in insert mode where any keys entered are inserted into the document but it can also run in command mode where keys are interpreted as commands.

Key Bindings

The default key binding to enter command mode is 'qq'. To exit command mode enter 'q'. Quink tries to make all the commands available through the default virtual keyboard on iOS. This makes it quicker to execute a command on a mobile device, but restricts the keys that are able to be bound to commands. To get round this restriction Quink uses the concept of secondary command maps. Commands in a secondary command map are accessed through a prefix key, effectively making them two letter commands. To execute these commands, enter the prefix key followed by the key for the command. If a prefix key is hit by mistake, hitting the same key again aborts the action.

Here are the prefix keys for the command maps. 

qq leave insert mode and enter command mode

q exit command mode and return to insert mode

f switch to font map

y switch to style map

p switch to para map

As an example, from insert mode to make the current selection bold the key sequence would be:

qqfb

When already in command mode to enter a numbered list:

pn

The toolbar

The Quink toolbar contains a number of tabs which are described in this section. Each of the commands available through the toolbar is shown along with its toolbar icon and command key binding. To use the key bindings Quink has to be in command mode which is achieved by hitting the command mode key ('qq' unless otherwise configured).

Style

The commands in this tab are for creating headers and formatting paragraphs. The key bindings are all prefixed by 'y'. Most of the commands in this map use the relative placement of the key on the keyboard to indicate the function that is to be carried out. The heading commands (H1 - H6) use the first 6 keys on the top row of the virtual keyboard. It would have been more obvious to use the number keys, but that would mean a keyboard switch in the virtual keyboard.

Icon Key Description
yq create an H1 heading
yw create an H2 heading
ye create an H3 heading
yr create an H4 heading
yt create an H5 heading
yy create an H6 heading
ya create a normal paragraph
yf create preformatted text

Font

Commands under the font tab allow the user to change font. The prefix key is 'f'.

Icon Key Description
fb make the selection bold
fi make the selection italic
fu underline the selection
fs strike through the selection
fa make the selection super script (Upper case 'A' looks a bit like an up arrow)
fv make the selection sub script (Upper case 'V' looks a bit like a down arrow)

Edit

Para

Para commands allow the insertion and manipulation of lists and paragraphs. The prefix key is 'p'.

Icon Key Description

The toolbar buttons will reflect the state of the document at the text insertion point. E.g. if the current selection is bold, the toolbar's bold button will show as active in both the misc and font tabs.

The Command Status Bar

Quink provides information about the state of the document at the text insertion point through the toolbar and also via the command status bar. The command status bar is a small display area in the top right of the page which displays icons appropriate to the document's state. Most of the icons are those familiar from the buttons on the editing toolbar.

In addition there are two extra icons. These additional icons display Quink's own internal state rather than the state of the document.

Icon Key Description
qq indicates that Quink is in command mode
v indicates that navigation will create a selection
indicates that the document contains unsaved changes
indicates that auto save has been switched off due to errors and that there are unsaved changes
